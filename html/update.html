AutoGMeet+ Updated!

What's New?

Version 1.6
[Bug] Extension crashes when calculating meeting duration
Do not calculating meeting duration when there is no meetings.
Version 1.5
[Bug] Crash when start time is not dateTime object, but date instead
For some meeting placeholder if meeting start time was a date object instead of dateTime, notifications were failing with crash.
[Bug] Join before did not worked. Actually join just in-time was applied.
Disregarding the setting Join Before, extension joined the meeting just in-time. Now setting will be taken into account on joining.
[Bug] In case meeting is too long, it would be joined twice.
Take meeting duration into account when parsing browser history.
Version 1.4
[Bug] Multiple tabs were opened for a meeting
Bug was still there even after fix in 1.3. Now it is solved.
[Bug] Join before did not worked. Actually join just in-time was applied.
Disregarding the setting Join Before, extension joined the meeting just in-time. Now setting will be taken into account on joining.
Version 1.3
[Bug] Multiple tabs were opened for a meeting
No more multiple tabs will be opened for a meeting.
Version 1.2
[Feature] Configure time when to join the meeting!
Chose between values (-2, -1, 0, 1), corresponding 2 or 1 minute before the start, 0 just in time, or be like a Royal - one minute late.
[Feature] Do not join declined meetings
Extension will not join declined meetings, or meeting that were declined by all others attendees.
[Bug] Extension goes inactive
Fixed the issue when extension's background service went inactive.
[Bug] Extension is spamming notifications
Fixed the issue that same notification was spammed multiple times.
